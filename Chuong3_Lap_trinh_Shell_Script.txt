Cú pháp ngôn ngữ Shell

Chuyển hướng Input  Output
• Trong Linux có thể chuyển kết quả output của lệnh này thành input của lệnh khác
• Có 3 loại chuyển hướng: > ; >>; <

• Symbol >: Lấy output cho lệnh tiếp theo
• $ ls > myfiles # ghi đè thông tin nếu đã tồn tại file
• Symbol >>: Output kết hợp với lệnh tiếp theo
• Syntax: Linux-command >> filename

• $ date >> myfiles # thêm thông tin vào phần cuối của file
• Symbol <
• Linux-command < filename

• $ cat < myfiles # Nhận đầu vào lệnh từ tập tin

Đường ống (pipeline)
• Ống dẫn là cách để kết nối output của một chương trình thành input của một
chương trình khác mà không dùng file tạm

• Ống dẫn thường dùng đễ thực thi nhiều hơn 1 câu lệnh trên cùng một dòng.
• Cú pháp : command1 | command2 | command3

Cú pháp ngôn ngữ Shell

Chi tiết lệnh xuất ra màn hình echo
Cú pháp: echo [options] [string, variables...]

• -e : cho phép hiểu những ký tự theo sau dấu trong chuỗi
• \n : xuống dòng
• -n : không in ký tự xuống dòng.

• \a : alert (tiếng chuông)
• \b : backspace
• \c : không xuống dòng

• \r : về đầu dòng
• \t : tab

Ví dụ:
echo -e “Cong \tnghe \tMMM“
echo -e “Cong \cnghe ma nguon mo"

Cú pháp ngôn ngữ Shell

Dấu nháy đôi “” và dấu nháy đơn ‘’
• Dấu nháy kép - “Double Quotes”: Nội dung trong “” được xem là chuỗi (ngoại trừ
$Biến hoặc ký tự đặc biệt \).
• Dấu nháy đơn – ‘Single quotes’: Nội dung trong ‘’ được xem là chuỗi, kể cả \ và $
• Dấu nháy huyền - `Back quotes` (ở phím ~): thực thi câu lệnh
Ví dụ:
o n=10
o echo 'So vua nhap la: $n' => In ra câu So vua nhap la: $n
o echo “So vua nhap la: $n” => In ra câu So vua nhap là: 10
o echo "Today is `date`". => In ra Today is Thứ Hai, 10 Tháng 10 năm 2023

Cấu trúc điều khiển
4.1 Lệnh kiểm tra điều kiện
• Trong ngôn ngữ lập trình shell, để kiểm tra điều kiện đúng hay sai ta sử
dụng lệnh test hoặc [ ]. Trả về 1 nếu FALSE, trả về 0 nếu TRUE.
• Shell cho kiểm tra trên 4 loại: Biến, Tập tin, Số, Chuỗi
Ví dụ về lệnh kiểm tra chuỗi
string1="HELLO Sir"

Ví dụ về lệnh kiểm tra số
var=10

string2="Hello"
[ $string1 > $string2 ] => trả về True (0)
Để kiểm tra kết qua ta dùng lệnh: echo $?
Lưu ý: Sau dấu [ và trước dấu ] phải có khoảng trắng

[ $var -eq 0 ] => FALSE (1)
Hoặc

test $var –eq 0 => false (1)

Xử lý chuỗi, văn bản
Lấy chuỗi con trong một chuỗi
• ${string:position}: Lấy chuỗi con từ vị trí position cho đến hết chuỗi.
• ${string:position:n}: trả về n ký tự trong chuỗi string bắt đầu ở vị trí position. Nếu
không có tham số n thì lấy từ position đến hết chuỗi.
Ví dụ: string=”this is a substring test”
substring=${string:10:9}

# lấy 9 ký tự trong chuỗi string từ vị trí thứ 10

echo $substring

=> kết quả là: substring

var="Welcome to the geekstuff“

echo ${var:15}

# lấy tất cả chuỗi từ vị trí 15

=> kết quả là: geekstuff

6

Xử lý chuỗi, văn bản
• Lấy n ký tự đầu chuỗi:

${string:0:n}

• Lấy n ký tự cuối chuỗi:

${string: -n}

• Lấy chiều dài của chuỗi:

${#string}

• Thay thế chuỗi:

#Lưu ý: có 1 khoảng trắng trước dấu -

${<string>/<string_to_replace>/<new_string>}

Ví dụ: string="Hello, Sir“
${string/Sir/Madam}
• Thay thế tất cả các chuỗi tìm kiếm được (replace all)
${<string>//<string_to_replace>/<new_string>}
Ví dụ: string=“Xanh Trắng Đen“
${string// /-}

=> kết quả Xanh-Trắng-Đen

6

Xử lý chuỗi, văn bản
• Lấy subtring bên phải của string. Nếu lấy chữ cuối thì substring là khoảng trắng.

${string##*substring}
Ví dụ: filename=“index.html”
echo ${filename##*.}

=> kết quả: html

• Trả về chuỗi bên phải của substring (xóa hết chuỗi bên trái kể cả substring). Nếu
xóa chữ đầu tiên thì substring là khoảng trắng.
${string#*substring}

Ví dụ: string=“Hi! How are you.”
echo ${string#*How}

=> kết quả: are you.

echo ${string#* }

=> kết quả: How are you.
